Bytecode = P-code 가상의 기계어[기계어와 유사]
JVM { Bytecode Interpreter Player Engine } 컴파일러 = javac.exe JVM = java.exe

하이브리드 방식을 취한 이유

1. 컴파일 방식의 단점
- os별로 컴파일을 해야 함 -> 실행 방식이 다르기 때문에
2. 인터프리터 방식의 단점
- 실행할 때 마다 소스파일이 있어야 함 -> 소스파일 공개됨
- 실행할 때 마다 문법 검사 -> 실행속도 느림
- 실행하기 전에는 문법 오류를 알 수 없음
3. 기계어와 흡사 -> 소스를 실행하기 전보다 실행속도(명령어 해석속도)가 빠르다
4. 한 번만 컴파일 하면 여러 os에서 실행할 수 있다
